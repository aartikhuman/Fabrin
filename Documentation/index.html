Introduction

Fabrin is a state-of-the-art React eCommerce template meticulously crafted for luxury boutiques, fashion houses, and premium lifestyle brands. Built with a "Mobile-First" approach, it offers a seamless shopping experience across all devices.

Ultra Performance

Optimized with Vite and modern React 18 standards for lightning-fast load times.

Design System

Meticulously organized CSS/Tailwind system for easy branding and consistency.

User Experience

Smooth Framer Motion animations and intuitive flows optimized for conversion.

Ready to Scale

Clean TypeScript architecture makes it easy for developers to extend functionality.

Key Features

  • Vite + React 18 + TypeScript
  • Framer Motion for high-end animations
  • Multi-step professional checkout flow
  • Dynamic product filtering system
  • Wishlist and Cart state management
  • Unique brand-reveal preloader
  • Retina ready & fully responsive layout

Requirements

Before you begin, ensure you have the following installed on your system:

Requirement Recommended Version
Node.js v18.0.0 or higher (Latest LTS recommended)
Package Manager npm v8+ or Yarn v1.22+
Browser Chrome, Edge, Safari, or Firefox (Modern versions)

Installation Guide

Follow these simple steps to get the environment ready:

1. Extract the Files

Extract the fabrin-v1.0.zip package to your desired project directory.

2. Install Dependencies

Open your terminal/command prompt and navigate to the project root:

cd fabrin
npm install

3. Running Development Server

Start the development environment to see changes in real-time:

npm run dev

The application will be available at: http://localhost:5173

4. Creating Production Build

When you are ready to deploy, generate the optimized production files:

npm run build

This will create a dist/ folder containing minimized assets.

Folder Structure

Fabrin follows a clean and logical industry-standard organization:

/app
  /components
    /layout      - Navigation, Header, Footer
    /ui          - Global UI elements (Preloader, Toasts)
    /product     - Cards, Gallery, Product-specific UI
    /category    - Filtering and Sidebar components
  /context       - Global state (Auth, Cart, Wishlist)
  /data          - Product and static content data
  /routes        - Page views (index, shop, about, etc.)
  /types         - TypeScript interfaces and types
  /utils         - Helper functions and animations
/public
  /assets        - High-resolution images and icons
/Documentation   - You are here!

Routing System

Routing is handled by React Router. All routes are centrally defined in app/routes.ts.

Dynamic Routing

Individual product pages are handled dynamically using the product ID:

route("product/:id", "routes/product.$id.tsx")

Checkout Flow

The checkout is a structured multi-step flow for better conversion:

  • Shipping Details → checkout.shipping.tsx
  • Payment Method → checkout.payment.tsx
  • Final Review → checkout.review.tsx

Customization Guide

Updating the Logo

The main logo text is defined in app/components/layout/Header.tsx. If you wish to use an image logo, replace the text Link with an <img /> tag.

Brand Colors & Typography

Global styling is handled via Tailwind CSS. You can modify colors and font families in:

  • tailwind.config.js - Configuration for colors and fonts.
  • app/app.css - Custom utility variables.

Managing Product Data

All products are stored in app/data/products.ts. Simply update this file to add, remove, or edit products for the entire site.

Pro Tip: When adding new pages, remember to register the route in app/routes.ts to make it accessible.
